# ToyEngine 着色器设计规范 (v1.0)

## 1. 资源绑定组 (Bind Groups) 规范

为了实现高效的资源切换和模块化设计，我们将绑定组按 **更新频率** 分为以下四个层级：

| 绑定组 | 频率 | 典型内容 |
| :--- | :--- | :--- |
| **@group(0)** | **全局级 (Frame)** | 摄像机 (View/Proj), 全局光照 (Global Lighting), 时间, 环境贴图 |
| **@group(1)** | **通道级 (Pass)** | 阴影贴图 (Shadow Map), 深度图, 屏幕空间反射 (SSR) 数据 |
| **@group(2)** | **材质级 (Material)** | 基础颜色纹理, 法线贴图, PBR 参数, 采样器 |
| **@group(3)** | **实例级 (Instance)** | 存储缓冲区 (Storage Buffer) 中的模型矩阵列表 (如果使用 SBO) |

---

## 2. 顶点输入 (Vertex Input) 规范

为了统一网格处理流程，所有自定义着色器应遵循以下 `@location` 分配规则：

### 2.1 顶点属性 (Per-Vertex)
| Location | 属性名 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| `@location(0)` | `position` | `vec3<f32>` | 模型空间位置 |
| `@location(1)` | `normal` | `vec3<f32>` | 模型空间法线 |
| `@location(2)` | `uv` | `vec2<f32>` | 第一套纹理坐标 |
| `@location(3)` | `color` | `vec4<f32>` | 顶点色 (可选) |

### 2.2 实例属性 (Per-Instance)
*注：通常用于不支持 Storage Buffer 的平台或简单实例化。*
| Location | 属性名 | 类型 | 说明 |
| :--- | :--- | :--- | :--- |
| `@location(5..8)` | `model_matrix` | `mat4x4<f32>` | 占用 4 个 location |

## 3. 模块化系统架构 (Blender 风格)

着色器文件组织结构如下：

```text
assets/shaders/
├── core/                # 核心函数库 (禁止包含 @vertex/@fragment 入口)
│   ├── lighting.wgsl    # 漫反射、高光、PBR 计算
│   ├── shadow.wgsl      # 阴影采样、阴影系数计算
│   ├── common.wgsl      # 常用工具函数 (颜色转换、坐标变换)
│   └── input.wgsl       # 核心结构体定义 (VertexInput, CameraUniform)
└── custom/              # 业务/自定义着色器
    ├── diffuse_only.wgsl
    └── pbr_standard.wgsl
```

### 3.1 引用原则
1.  **Core 优先**：自定义着色器必须引用 `core/` 中的结构体和函数。
2.  **职责分离**：`core/` 负责 **"怎么算"** (计算逻辑)，`custom/` 负责 **"怎么接"** (输入输出)。

## 4. 示例代码规范

### 核心结构体 (core/input.wgsl)
```wgsl
struct VertexInput {
    @location(0) position: vec3<f32>,
    @location(1) normal: vec3<f32>,
    @location(2) uv: vec2<f32>,
};

struct CameraUniform {
    view_proj: mat4x4<f32>,
    view_pos: vec3<f32>,
};
```

### 核心函数 (core/lighting.wgsl)
```wgsl
fn calculate_diffuse(normal: vec3<f32>, light_dir: vec3<f32>) -> f32 {
    return max(dot(normal, light_dir), 0.0);
}
```

### 自定义着色器 (custom/my_shader.wgsl)
```wgsl
// 注意：WGSL 目前不支持原生的 #include，引擎层需实现预处理或使用模块化库
// 假定引擎支持 import 语法
#import core::input::{VertexInput, CameraUniform}
#import core::lighting::calculate_diffuse

@group(0) @binding(0) var<uniform> camera: CameraUniform;

@vertex
fn vs_main(in: VertexInput) -> ... {
    // 逻辑
}
```
