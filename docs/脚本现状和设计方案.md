# 脚本系统现状与设计方案

## 一、现状分析

### 1.1 当前实现

**WASM 运行时**: `crates/engine-app/src/wasm_runtime.rs`

使用 **Wasmer 4.x** (cranelift 编译器):

```rust
pub struct WasmRuntime {
    store: Store,
    instance: Option<Instance>,
}
```

### 1.2 当前接口

**参数传递方式**: 裸 i32 位模式转换

```rust
// 引擎侧 - 将 f32 转为 i32 位模式
fn f32_to_i32_bits(f: f32) -> i32 {
    i32::from_le_bytes(f.to_le_bytes())
}

// 传递的参数
- dt: delta time (f32 as i32)
- frame_count: frame counter (u32 as i32)
- radius/orbit_radius: orbit radius (f32 as i32)
- height: camera height (f32 as i32)
- speed: rotation speed (f32 as i32)
- input_mask: key state bitmask (u8 as i32)
```

**返回值**: 三次独立函数调用获取 xyz

```rust
get_camera_x() -> i32 -> f32
get_camera_y() -> i32 -> f32
get_camera_z() -> i32 -> f32
```

### 1.3 现状问题

| 问题 | 描述 | 严重程度 |
|------|------|----------|
| 无脚本 Trait | 缺少通用脚本接口 trait | 高 |
| 紧耦合 | 脚本只能控制相机，引擎侧硬编码调用 | 高 |
| 通信困难 | 通过裸 i32 位模式传递参数，无结构化数据 | 中 |
| 无生命周期 | 缺少 init/update/shutdown 钩子 | 中 |
| 无 ECS 交互 | 脚本无法访问或修改 ECS World | 高 |
| 依赖混乱 | workspace 声明 wasmtime，实际使用 wasmer | 低 |

### 2.2 核心 Trait 设计

#### ScriptError

```rust
#[derive(Error, Debug)]
pub enum ScriptError {
    #[error("Load error: {0}")]
    Load(String),
    #[error("Runtime error: {0}")]
    Runtime(String),
    #[error("Function not found: {0}")]
    FunctionNotFound(String),
    #[error("Call error: {0}")]
    Call(String),
}
```

#### ScriptContext

```rust
/// 脚本上下文 - 脚本访问引擎的接口
pub trait ScriptContext: Send + Sync {
    /// 获取 ECS World 引用
    fn world(&self) -> &World;
    
    /// 获取可变 World 引用
    fn world_mut(&mut self) -> &mut World;
    
    /// 获取资源（只读）
    fn get_resource<R: Resource>(&self) -> Option<&R>;
    
    /// 获取资源（可变）
    fn get_resource_mut<R: Resource>(&mut self) -> Option<&mut R>;
    
    /// 查询实体
    fn query<Q: QueryType>(&self) -> QueryResults<Q>;
    
    /// 日志输出
    fn log(&self, level: &str, message: &str);
    
    /// 获取时间信息
    fn delta_time(&self) -> f32;
    fn total_time(&self) -> f64;
    fn frame_count(&self) -> u32;
}
```

#### Script

```rust
/// 脚本 Trait - 所有脚本实现的接口
pub trait Script: Send + Sync {
    /// 初始化脚本
    fn init(&mut self, ctx: &mut dyn ScriptContext) -> Result<(), ScriptError>;
    
    /// 每帧更新
    fn update(&mut self, ctx: &mut dyn ScriptContext) -> Result<(), ScriptError>;
    
    /// 关闭时调用
    fn shutdown(&mut self, ctx: &mut dyn ScriptContext) -> Result<(), ScriptError>;
    
    /// 脚本名称（用于调试）
    fn name(&self) -> &str;
}
```

### 2.3 WASM 运行时设计

#### WasmScriptHost

```rust
/// WASM 脚本主机 - 桥接 Wasmer 和 Script Trait
pub struct WasmScriptHost {
    store: Store,
    module: Option<Module>,
    instance: Option<Instance>,
    context: WasmScriptContext,
}

impl WasmScriptHost {
    pub fn new() -> Result<Self, ScriptError>;
    pub fn load(&mut self, path: &Path) -> Result<(), ScriptError>;
    pub fn init(&mut self, ctx: &mut dyn ScriptContext) -> Result<(), ScriptError>;
    pub fn update(&mut self, ctx: &mut dyn ScriptContext) -> Result<(), ScriptError>;
    pub fn shutdown(&mut self, ctx: &mut dyn ScriptContext) -> Result<(), ScriptError>;
}
```

### 2.4 ScriptManager

```rust
/// 脚本管理器 - 管理所有脚本的生命周期
pub struct ScriptManager {
    scripts: Vec<Box<dyn Script>>,
    runtime: Option<WasmScriptHost>,
}

impl ScriptManager {
    pub fn new() -> Self;
    
    /// 加载脚本
    pub fn load_script(&mut self, path: &Path) -> Result<(), ScriptError>;
    
    /// 初始化所有脚本
    pub fn init_all(&mut self, ctx: &mut dyn ScriptContext);
    
    /// 更新所有脚本
    pub fn update_all(&mut self, ctx: &mut dyn ScriptContext);
    
    /// 关闭所有脚本
    pub fn shutdown_all(&mut self, ctx: &mut dyn ScriptContext);
}
```

## 三、实施步骤

1. 创建 `crates/engine-script` crate

3. 实现变更追踪

## 五、预期收益

1. **解耦** - 引擎不再硬编码游戏逻辑
2. **可扩展** - 可添加 JS/Lua 等其他脚本运行时
3. **可测试** - Mock ScriptContext 进行单元测试
4. **规范化** - 统一的脚本开发接口
5. **ECS 交互** - 脚本可访问和修改 ECS World
