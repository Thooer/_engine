# ToyEngine 开发 TODO / 里程碑路线图

> 本文是**全局开发路线图**，从“能跑起来”到“可玩 Demo / 编辑器雏形”的分阶段目标。
> 每一阶段尽量保证：**可以单独运行，有一个小 Demo，可以作为 Checkpoint 打 Tag**。

---

## 阶段 0：项目初始化 & 底层准备

- [ ] **仓库 & CI**
  - [ ] 初始化 Git 仓库 / Workspace 结构
  - [ ] 配置基础 CI（格式检查 + 构建）
  - [ ] 提交 `Cargo.toml` / `Cargo.lock` / `.gitignore`
- [ ] **基础依赖确认**
  - [ ] 确认技术栈中核心库版本：`winit`、`wgpu`、`bevy_ecs`、`glam` 等
  - [ ] 能在本机成功 `cargo run` 一个空程序

**阶段目标**：环境能一键拉起，CI 通过，主程序可以空跑。

---

## 阶段 1：Platform + Core（窗口 & 主循环骨架）

- [ ] **Platform 层**
  - [x] 使用 `winit` 打开窗口，正确处理关闭事件
  - [x] 整合 `raw-window-handle`，为后续 `wgpu` 做准备
- [ ] **Core / Application**
  - [x] 定义引擎生命周期：`init → update(dt) → shutdown`
  - [x] 实现主循环（固定帧 / 变步长，至少选一种）
  - [x] 集成日志系统（`tracing`），在主循环中打印基础信息

**阶段目标**：可以打开一个窗口，主循环稳定运行，日志输出可观测。

---

## 阶段 2：基础渲染通路（RHI + Renderer v0）

- [ ] **RHI / wgpu 初始化**
  - [ ] 使用 `wgpu` 创建 `Instance/Adapter/Device/Queue/Swapchain`
  - [ ] 清屏渲染：能以指定颜色 Clear 屏幕
- [ ] **绘制基础图元**
  - [ ] 使用 WGSL 着色器画出一个三角形 / 简单几何体
  - [ ] 支持窗口大小变化时正确重建 Swapchain

**阶段目标**：一个稳定的“清屏 + 三角形” Demo，证明渲染管线工作正常。

---

## 阶段 3：ECS 场景 & 渲染集成

- [ ] **ECS（`bevy_ecs`）引入**
  - [ ] 定义基础组件：Transform、Renderable 等
  - [ ] 定义基础系统：Transform 更新、渲染收集
- [ ] **Renderer v0 与 ECS 集成**
  - [ ] 从 ECS 中读取可渲染实体，统一提交到渲染层
  - [ ] 支持多个简单实体（不同位置/颜色）的渲染

**阶段目标**：在场景中放置多个实体，通过 ECS 更新，在屏幕上正确渲染。

---

## 阶段 4：资源系统 v0（Asset Loading）与基础 IO

- [ ] **File/IO & VFS**
  - [ ] 抽象统一的文件访问接口（本地文件即可）
  - [ ] 预留 VFS / 远程存储接口（可以先不实现）
- [ ] **资源加载 v0**
  - [ ] 使用 `gltf` / `image` 等库，从磁盘加载模型和纹理
  - [ ] 简单缓存（同一资源不会重复加载）
  - [ ] 支持从配置中指定要加载的场景/模型

**阶段目标**：能通过配置指定 glTF 模型 / 纹理，并在场景中渲染出来。

---

## 阶段 5：输入系统 & 相机控制 & 调试 UI v0

- [ ] **输入系统 v0**
  - [ ] 使用 `winit` 键鼠事件，建立最小的 Input 抽象
  - [ ] 支持“按键状态查询”与“按键触发事件”两种模式
- [ ] **相机控制**
  - [ ] 实现自由飞行相机（WASD + 鼠标）
  - [ ] 或实现轨道相机（围绕目标旋转）
- [ ] **Debug UI v0**
  - [ ] 集成 `egui`（`egui-winit` + `egui-wgpu`）
  - [ ] 显示 FPS / 渲染对象数量等基础信息
  - [ ] 提供几个简单开关（例如：开关 VSync / 切换渲染模式）

**阶段目标**：可以在场景中自由飞行查看模型，并通过 Debug UI 观察和切换基本状态。

---

## 阶段 6：物理 & 音频 & 更丰富的 Playground

- [ ] **物理系统 v0（rapier）**
  - [ ] 引入 `rapier3d` / `rapier2d`，创建物理世界
  - [ ] 将 ECS Transform 与物理刚体同步（读/写 两个方向）
  - [ ] 简单碰撞检测和地面/重力 Demo
- [ ] **音频系统 v0（kira）**
  - [ ] 在启动时播放背景音乐
  - [ ] 触发特定事件时播放音效（例如：物体落地声）
- [ ] **Playground 场景**
  - [ ] 若干物体掉落在地面，有简单交互（例如按键生成新物体）
  - [ ] 使用 Debug UI 切换不同测试场景

**阶段目标**：有一个物理+音频+相机的“小游乐场”场景，可用来验证底层各系统协作。

---

## 阶段 7：存档 / 序列化 & 场景配置

- [ ] **序列化基础**
  - [ ] 使用 `serde` + `bincode` / `ron` 对场景配置进行序列化
  - [ ] 至少支持：实体的 Transform、模型引用、简单属性
- [ ] **存档与加载**
  - [ ] 支持从文件加载一个场景配置并实例化
  - [ ] 支持在运行时保存当前场景到文件（调试用）

**阶段目标**：可以用数据文件描述一个场景，运行时加载/保存，实现最基础的“关卡数据驱动”。

---

## 阶段 8：脚本系统 v0（WASM）与 Gameplay 雏形

- [ ] **WASM 脚本运行时**
  - [ ] 选择 WASM 容器（如 `wasmtime` / `wasmer` 等）
  - [ ] 支持加载一个 WASM 模块并调用其中的更新函数
  - [ ] 提供基础绑定（时间、Transform 读写等）
- [ ] **Gameplay 框架 v0**
  - [ ] 定义一些通用组件（如 Health、Trigger、简单状态机等）
  - [ ] 用 WASM 编写一个简单交互逻辑（例如：靠近门自动打开）

**阶段目标**：通过脚本实现一个“小玩法 Demo”，验证“引擎驱动 + 脚本编写逻辑”的工作流。

---

## 阶段 9：编辑器雏形 & 工具化

- [ ] **Editor Shell v0**
  - [ ] 基于 `egui` 实现：场景层级视图 + Inspector（属性面板）
  - [ ] 支持选中实体并实时修改 Transform / 组件属性
- [ ] **基础工具链**
  - [ ] 资源浏览器：列出已加载资源，支持简单刷新/重载
  - [ ] DebugDraw：物理碰撞体、摄像机视锥等可视化

**阶段目标**：形成一个“编辑器+运行时一体”的雏形，可以在编辑器里布置物体并立即预览。

---
