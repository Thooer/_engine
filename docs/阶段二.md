## 二、设计方案

### 1. 异步资源加载器 (P0 - 大工作量)

**目标**: 异步加载资源，不阻塞主线程

```rust
// engine-core/src/resources/mod.rs

use bevy_ecs::prelude::Resource;
use std::path::PathBuf;
use std::sync::Arc;
use tokio::sync::mpsc;

/// 资源句柄 - 用于异步加载
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
pub struct AssetHandle(Uuid);

/// 资源状态
#[derive(Debug, Clone, Copy)]
pub enum LoadState {
    NotLoaded,
    Loading,
    Loaded,
    Failed,
}

/// 资源元数据
pub struct AssetMetadata {
    pub handle: AssetHandle,
    pub path: PathBuf,
    pub load_state: LoadState,
}

/// 资源服务器 - 核心资源管理
#[derive(Resource)]
pub struct AssetServer {
    /// 资源缓存
    cache: DashMap<AssetHandle, Arc<dyn Any + Send + Sync>>,
    /// 元数据
    metadata: DashMap<AssetHandle, AssetMetadata>,
    /// 加载请求通道
    load_tx: mpsc::Sender<LoadRequest>,
}

/// 加载请求
pub struct LoadRequest {
    pub path: PathBuf,
    pub handle: AssetHandle,
    pub loader_type: LoaderType,
}

pub enum LoaderType {
    Model,
    Texture,
    Material,
}
```

**设计要点**:
- 使用 `tokio` 异步加载
- 使用 `DashMap` (from `dashmap`) 并发安全的 HashMap
- 资源句柄用 UUID，支持引用计数

### 2. 渲染图抽象 (P1 - 大工作量)

**目标**: 灵活的渲染管线配置

```rust
// engine-renderer/src/render_graph/mod.rs

use bevy_ecs::prelude::World;
use std::any::TypeId;

/// 渲染节点
pub trait RenderNode: Send + Sync {
    fn name(&self) -> &str;
    fn execute(&self, ctx: &mut RenderContext);
    fn inputs(&self) -> Vec<Port>;
    fn outputs(&self) -> Vec<Port>;
}

/// 渲染端口
#[derive(Debug, Clone, Hash, PartialEq, Eq)]
pub struct Port {
    pub name: String,
    pub type_id: TypeId,
}

/// 渲染图
pub struct RenderGraph {
    nodes: HashMap<String, Box<dyn RenderNode>>,
    edges: Vec<(Port, Port)>,
}

impl RenderGraph {
    pub fn new() -> Self { ... }
    
    /// 添加节点
    pub fn add_node(&mut self, name: &str, node: impl RenderNode + 'static) { ... }
    
    /// 连接节点
    pub fn add_edge(&mut self, from: (&str, &str), to: (&str, &str)) { ... }
    
    /// 执行渲染图
    pub fn execute(&self, world: &World, ctx: &mut RenderContext) { ... }
}

/// 渲染上下文
pub struct RenderContext {
    pub device: wgpu::Device,
    pub queue: wgpu::Queue,
    pub encoder: wgpu::CommandEncoder,
    /// 中间渲染目标
    pub textures: HashMap<String, wgpu::TextureView>,
}
```

**当前 Pass 可迁移为节点**:
- `MeshForwardPass` → `MeshForwardNode`
- `LinePass` → `LineNode`

### 3. 场景序列化 (P1 - 中工作量) - 反射方案

**目标**: 使用 Bevy 反射实现场景序列化，支持编辑器

> **重要**：选择反射而非手动实现，因为：
> - 组件数量即将增加
> - 需要编辑器 UI
> - 手动方案每次改组件都需要改 2 处（enum + match）

---

#### 3.1 技术方案

使用 `bevy_reflect` + `DynamicScene` 实现：

```rust
// 1. 组件定义 - 只需要两个 derive
#[derive(Component, Reflect, Default)]
#[reflect(Component)]  // 关键：让反射系统知道这是组件
pub struct MyComponent {
    pub value: f32,
    #[reflect(skip_serializing)]  // 可选：跳过某些字段
    pub runtime_only: String,
}

// 2. 序列化 - 一行搞定
let scene = DynamicScene::from_world(&world);
let ron = scene.serialize(&type_registry).unwrap();

// 3. 反序列化 - 一行搞定
let mut entity_map = EntityHashMap::default();
scene.write_to_world(&mut world, &mut entity_map).unwrap();
```

#### 3.4 场景加载器实现

```rust
// engine-core/src/scene/mod.rs

use bevy_ecs::prelude::*;
use bevy_reflect::{DynamicScene, TypeRegistry, TypeRegistryExt};
use std::collections::HashMap;
use std::path::Path;

/// 场景加载器
#[derive(Resource)]
pub struct SceneLoader;

impl SceneLoader {
    /// 从文件加载并 spawn 到 World
    pub fn load_and_spawn(path: &str, world: &mut World) -> Result<(), SceneError> {
        // 读取 RON 文件
        let content = std::fs::read_to_string(path)
            .map_err(SceneError::Io)?;
        
        // 获取类型注册表
        let type_registry = world.resource::<AppTypeRegistry>();
        let type_registry = type_registry.read();
        
        // 反序列化场景
        let mut deserializer = ron::Deserializer::from_str(&content)
            .map_err(|e| SceneError::Parse(e.to_string()))?;
        
        let dynamic_scene = DynamicScene::deserialize(&mut deserializer, &type_registry)
            .map_err(|e| SceneError::Deserialize(e.to_string()))?;
        
        // Spawn 到 World
        let mut entity_map = EntityHashMap::default();
        dynamic_scene.write_to_world(world, &mut entity_map)
            .map_err(|e| SceneError::Spawn(e.to_string()))?;
        
        Ok(())
    }
    
    /// 从 World 保存到文件
    pub fn save_and_write(world: &mut World, path: &str) -> Result<(), SceneError> {
        let type_registry = world.resource::<AppTypeRegistry>();
        
        // 创建动态场景
        let scene = DynamicScene::from_world(world);
        
        // 序列化
        let content = scene.serialize(type_registry)
            .map_err(|e| SceneError::Serialize(e.to_string()))?;
        
        // 写入文件
        std::fs::write(path, content)
            .map_err(SceneError::Io)?;
        
        Ok(())
    }
}

/// 场景错误
#[derive(Debug)]
pub enum SceneError {
    Io(std::io::Error),
    Parse(String),
    Serialize(String),
    Deserialize(String),
    Spawn(String),
}

impl std::fmt::Display for SceneError {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        match self {
            Self::Io(e) => write!(f, "IO error: {}", e),
            Self::Parse(e) => write!(f, "Parse error: {}", e),
            Self::Serialize(e) => write!(f, "Serialize error: {}", e),
            Self::Deserialize(e) => write!(f, "Deserialize error: {}", e),
            Self::Spawn(e) => write!(f, "Spawn error: {}", e),
        }
    }
}
```

#### 3.9 注意事项

1. **模型加载**：模型文件 (.glb) 需要通过 `ModelLoader` 加载，场景文件只包含 `MeshRenderable` 引用。

2. **Enum 序列化**：如果组件包含 enum，需要确保 enum 也实现了 `Reflect`。

3. **Entity ID**：序列化时会包含实体 ID，反序列化时默认生成新 ID（通过 `EntityHashMap` 映射）。

**已实现的功能**:

**代码特点**:

    Transform { ... },

### 我们的设计与现有 Demo 对比

| 特性 | scene_demo.rs (现状) | 阶段二设计 (目标) | 优势 |
|------|---------------------|-----------------|------|
| 模型加载 | 同步阻塞 | 异步非阻塞 | 不卡顿主线程 |
| 资源管理 | HashMap 手动管理 | AssetServer + Handle | 引用计数、依赖追踪 |
| 场景表示 | 硬编码 Rust | RON 文件 | 可编辑、可序列化 |
| 场景加载 | 编译时决定 | 运行时加载 | 动态关卡切换 |
| 资源缓存 | 简单 HashMap | 智能缓存 + 热重载 | 修改即时生效 |
| 渲染管线 | 硬编码 Pass 顺序 | RenderGraph 节点化 | 动态增删渲染阶段 |

---

### 具体超越点

1. **异步资源加载**
   - Demo: `ModelLoader::load_gltf()` 同步阻塞
   - 设计: `AssetServer::load_async()` 后台线程加载

2. **资源句柄系统**
   - Demo: 字符串 ID (`mesh_id: "monkey"`)
   - 设计: `AssetHandle(UUID)` + 引用计数

3. **场景序列化**
   - Demo: 代码中硬编码实体
   - 设计: `.ron` 文件，可保存/加载

4. **渲染管线**
   - Demo: `MeshForwardPass` → `LinePass` 固定顺序
   - 设计: 可配置渲染图，动态添加/删除/重排序

---

### 结论

我们的设计在以下方面超越现有 Demo:
- **资源加载**: 从同步阻塞 → 异步非阻塞
- **资源管理**: 从手动字符串ID → 智能Handle系统
- **场景系统**: 从代码硬编码 → 文件序列化
- **渲染管线**: 从固定Pass → 可配置RenderGraph

现有 Demo 是**概念验证 (PoC)**，我们的设计是**生产级架构**。
